# Todo App - Blueprint Demo

A complete, runnable todo application generated by the Blueprint Scaffolder.

## Quick Start

```bash
# Install dependencies
pip install -r requirements.txt

# Run the app
python app.py

# Open in browser
# http://localhost:5000
```

## What's Included

- **REST API** - Full CRUD operations for tasks
- **Web Frontend** - Clean, responsive UI (no build step needed)
- **JSON Storage** - Simple file-based persistence
- **Health Check** - `/health` endpoint for monitoring

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/tasks` | List all tasks |
| POST | `/api/tasks` | Create a task |
| GET | `/api/tasks/<id>` | Get a task |
| PUT | `/api/tasks/<id>` | Update a task |
| DELETE | `/api/tasks/<id>` | Delete a task |
| POST | `/api/tasks/<id>/toggle` | Toggle completion |

## Example API Usage

```bash
# Create a task
curl -X POST http://localhost:5000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "Buy groceries", "priority": "high"}'

# List all tasks
curl http://localhost:5000/api/tasks

# Toggle task completion
curl -X POST http://localhost:5000/api/tasks/1/toggle

# Delete a task
curl -X DELETE http://localhost:5000/api/tasks/1
```

## Generated By

This app was generated using **Constraint-First Development**:

- Storage: JSON (upgradeable to SQLite or PostgreSQL)
- Backend: Flask
- Auth: None (add with `auth_basic` block)
- Blocks used: `backend_flask`, `storage_json`, `crud_routes`

To regenerate with different options:

```bash
cd blueprints
python blueprint.py create "todo app with auth" --output my_todo
```

## Upgrading

- **Add Auth**: Include `auth_basic` block
- **Use SQLite**: Switch to `storage_sqlite` block  
- **Add API Docs**: Add OpenAPI/Swagger annotation
- **Deploy**: Use `docker_basic` block for containerization
